# Research Process Workflow

**Goal:** Standardize the research lifecycle from request to delivery.

---

## üîÑ Workflow Steps

1. **Request Intake (Notion)**
   - New item created in "Research Hub".
   - **Status:** `To Do`.
   - **Properties:** Topic, Category, Deadline filled.

2. **Wait for Approval**
   - Project Manager reviews request.
   - **Status:** Move to `In Progress`.

3. **Preliminary Scan (AI)**
   - Run **Quick Market Scan** prompt.
   - Paste prompt output into the "AI Summary" property in Notion.

4. **Deep Dive & Validation**
   - Identify 2-3 key questions from the scan.
   - Use Perplexity Pro to verify data points.
   - Add source URLs to the "Source URL" property.

5. **Reporting (Canva/PPT)**
   - Copy validated data into the standardized template.
   - Export as PDF.
   - Upload to Notion or link in the "Deliverable" property.

6. **Review & Archive**
   - **Status:** Move to `Done`.
   - Tag with relevant projects.
